## üéì Learning Paths

### Path 1: "I want to play SubSpace"
1. Read [QUICKSTART.md](QUICKSTART.md)
2. Install and run the game
3. Check [SUBSPACE_README.md](SUBSPACE_README.md) for features
4. Join the community (Discord coming soon)

### Path 2: "I want to contribute code"
1. Read [BUILD_GUIDE.md](BUILD_GUIDE.md) to set up your environment
2. Read [CONTRIBUTING_SUBSPACE.md](CONTRIBUTING_SUBSPACE.md) thoroughly
3. Study [ARCHITECTURE.md](ARCHITECTURE.md) to understand the codebase
4. Check [DEVELOPER_WORKFLOW.md](DEVELOPER_WORKFLOW.md) for workflows
5. Pick an issue and start coding!

### Path 3: "I want to create content"
1. Read [QUICKSTART.md](QUICKSTART.md) to understand the game
2. Read [CONTRIBUTING_SUBSPACE.md](CONTRIBUTING_SUBSPACE.md) content section
3. Study existing content in `dat/` directory
4. Create and submit your content!

### Path 4: "I want to use this for my own game"
1. Read [LICENSING_COMPLIANCE.md](LICENSING_COMPLIANCE.md) **CAREFULLY**
2. Read [SUBSPACE_PROJECT.md](SUBSPACE_PROJECT.md) to understand the codebase
3. Read [ARCHITECTURE.md](ARCHITECTURE.md) for technical details
4. Fork and customize while following GPL requirements!

### Path 5: "I'm evaluating this for business use"
1. Read [PROJECT_SUMMARY.md](PROJECT_SUMMARY.md) for overview
2. Read [LICENSING_COMPLIANCE.md](LICENSING_COMPLIANCE.md) for legal implications
3. Read [ARCHITECTURE.md](ARCHITECTURE.md) for technical assessment
4. Consult with a lawyer specializing in open source licensing

### Path 6: "I'm curious about the project"
1. Start with [PROJECT_SUMMARY.md](PROJECT_SUMMARY.md)
2. Read [SUBSPACE_VS_NAEV.md](SUBSPACE_VS_NAEV.md) for context
3. Skim [FEATURES_BRAINSTORM.md](FEATURES_BRAINSTORM.md) for cool ideas
4. Try [QUICKSTART.md](QUICKSTART.md) to play!
